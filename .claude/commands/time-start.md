# 任务
- 1.使用`能力`中定义的时间线搜索功能，对历史会话存档进行回忆检索
- 2.默认回忆范围为最近7天（从当前日期向前推7天）
- 3.若用户指定了关键词，则在搜索范围内按关键词匹配会话主题和摘要
- 4.若用户指定了自定义天数，则按用户指定的天数范围进行回忆
- 5.将搜索结果按时间倒序整理并呈现给用户

# 能力
- 位置:`.claude/abilities/TimeLine.md`
- 核心参考：「场景三：时间线搜索」

# 参数
- **关键词**（可选）：用于匹配会话主题或摘要的搜索关键词，支持模糊匹配。若不指定，则列出搜索范围内全部会话记录
- **天数**（可选）：自定义回忆的天数范围，默认值为`7`天。例如 `14` 表示回忆最近14天
- **追溯深度**（可选）：关联时间线的追溯层数，默认值为`3`层，最大不超过`5`层

# 执行流程
- 1.**确定时间范围**：使用 Bash 获取当前日期，根据参数计算搜索起止日期范围
- 2.**读取索引**：按日期从近到远，依次读取各日期文件夹下的 `INDEX.md`
- 3.**匹配检索**：在 INDEX.md 中匹配会话主题和摘要内容，命中后通过 wikilink 深入读取详情文件
- 4.**无关键词模式**：若用户未指定关键词，则列出搜索范围内所有 INDEX.md 中的索引条目
- 5.**降级策略**：若所有 INDEX.md 均无匹配结果，则对指定日期范围内的详情文件进行关键词搜索
- 6.**关联追溯**：若详情文件中包含「关联记忆时间线」章节，按指定追溯深度继续查看关联文件
- 7.**结果整理**：将搜索结果按时间倒序排列，标注匹配来源和文件路径

# 强制要求
- 严格按照`能力`文件中「场景三：时间线搜索」的全部约束和规则执行
- 强制遵循「先索引后详情」策略：始终先查看 INDEX.md，只有索引匹配时才通过 wikilink 深入读取详情文件
- 禁止直接遍历 `{YYYY-MM-DD}/TIMELINE/` 文件夹下所有详情文件进行全量扫描
- INDEX.md 无匹配则立即跳过该日期，不做多余读取
- 搜索结果必须包含：匹配内容摘要 + 文档完整相对路径
- 不得编造或猜测不存在的会话内容
- 路径基准强制为当前项目根目录下的 `.claude/timelines/`
- 关联时间线追溯最大深度为5层
- 强制使用 Bash 查看当前日期，不得凭记忆推断日期

# 使用示例
| 调用方式 | 说明 |
|---------|------|
| `/task-start` | 回忆最近7天所有会话记录 |
| `/task-start 登录模块` | 搜索最近7天中与"登录模块"相关的会话 |
| `/task-start 接口优化 14` | 搜索最近14天中与"接口优化"相关的会话 |
| `/task-start 部署 30` | 搜索最近30天中与"部署"相关的会话 |

# 注意事项
- 若指定天数范围内无任何会话存档，直接告知用户「该时间范围内无存档记录」
- 搜索结果较多时（超过5条），先展示索引摘要列表，让用户选择需要深入查看的具体条目
- 本命令的调用操作**不需要**记录到存档中，避免无限循环
- 与 [[task-end.md|task-end 命令]] 配合使用，形成「存档 → 回忆」的完整工作流闭环
- 搜索结果中每条记录必须附带完整文件路径，便于后续存档时建立关联记忆时间线
